## 1. 问题描述

### 1.1 发现的问题
当前的记忆系统依赖手动反思来捕获信号，存在以下问题：
- 手动反思容易遗漏重要信号
- 信号捕获不够及时
- 缺少结构化的信号记录机制
- 信号数据不够丰富，难以进行后续分析

### 1.2 根本原因
缺乏自动信号捕获机制，所有信号都需要手动记录。

---

## 2. 实施方案

### 2.1 方案设计
**创建 Signal Capture System**，在关键交互点自动捕获信号：

1. **任务完成信号** - 在任务完成时自动捕获
2. **工具使用信号** - 在工具使用时自动捕获
3. **用户反馈信号** - 在用户提供反馈时自动捕获

### 2.2 创建的组件

#### 1. `signal_capture.py` - 信号捕获系统

**核心类**: `SignalCapture`

**主要功能**:

1. **`capture_task_complete()`** - 捕获任务完成信号
   - 任务 ID 和标题
   - 任务成功/失败状态
   - 任务持续时间（毫秒）
   - 任务输出摘要
   - 任务输出质量

2. **`capture_tool_usage()`** - 捕获工具使用信号
   - 工具名称
   - 执行时间（毫秒）
   - 返回码
   - 输出长度
   - 执行成功/失败状态

3. **`capture_user_feedback()`** - 捕获用户反馈信号
   - 任务 ID
   - 用户满意度（满意/不满意）
   - 反馈文本

4. **`get_recent_signals()`** - 获取最近的信号
   - 可按信号类型过滤
   - 可限制返回数量
   - 按时间倒序排列

**信号存储**:
- 存储位置: `memory/signals/`
- 文件命名: `{signal_type}-{timestamp}.jsonl`
- 信号格式: JSONL，便于解析
