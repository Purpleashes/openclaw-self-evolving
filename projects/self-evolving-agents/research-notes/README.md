
# 研究文档模板使用指南

本目录包含标准化的研究文档模板，用于确保我们的研究笔记格式一致、结构清晰。

## 模板列表

### 1. RESEARCH_TEMPLATE.md - 基础研究模板
**适用场景**: 简短的研究笔记、快速分析

**包含部分**:
- 标题和元数据
- 项目概览
- 核心分析章节
- 关键发现
- 借鉴意义
- 后续行动计划
- 总结

### 2. RESEARCH_TEMPLATE_COMPREHENSIVE.md - 综合研究模板
**适用场景**: 深入的项目分析、技术调研、详细研究

**包含部分**:
- 标题和元数据（包含研究类型）
- 目录导航
- 详细概览（表格形式）
- 核心概念分析
- 功能模块深度解析
- 技术细节和架构设计
- 关键发现（带重要性和证据）
- 分阶段借鉴意义（短/中/长期）
- 详细行动计划（表格+步骤）
- 完整总结

### 3. ITERATION_TEMPLATE.md - 迭代文档模板
**适用场景**: 记录自我迭代的设计和实施

**包含部分**:
- 迭代编号和名称
- 问题陈述
- 改进想法
- 实施步骤
- 预期结果
- 评估方法

## 使用规范

### 命名规范
- 研究笔记: `day-XX-[简短描述].md`
- 迭代文档: `iteration-XXX.md`
- 评估文档: `iteration-XXX-evaluation.md`

### 元数据规范
所有文档开头必须包含：
```markdown
# [文档标题]

**日期**: YYYY-MM-DD
**任务**: [任务ID - 任务名称]
**优先级**: [P0/P1/P2/P3]
```

### 选择模板的原则

| 研究类型 | 推荐模板 | 理由 |
|---------|---------|------|
| 快速调研、初步分析 | RESEARCH_TEMPLATE.md | 简洁高效 |
| 深入项目分析、技术拆解 | RESEARCH_TEMPLATE_COMPREHENSIVE.md | 结构完整 |
| 自我迭代设计 | ITERATION_TEMPLATE.md | 针对性强 |

## 最佳实践

1. **先选模板再写作** - 根据研究深度选择合适的模板
2. **保持结构完整** - 不要随意删除模板中的章节
3. **及时更新元数据** - 确保日期、任务、优先级准确
4. **添加具体内容** - 避免空泛描述，要具体、有数据
5. **包含行动项** - 研究要有后续行动计划
